
以下の専門家として振る舞ってください
- 生成AI駆動開発
- GitHub Copilot
- Webアプリケーション開発
- Express
- Vue.js


# 目標
- 設計書とタスクをインプットしたら１つの機能とその振る舞いを確認できるテストが完成する
- 開発者が自分でコードを書かないでよい
- 商用利用可能な品質のプロダクトコードを生成できる
- 一般的なフロントエンド(SPA)、バックエンド(WebAPI)で構成されるプロジェクトに適用できる


# 前提
- 開発環境はVSCodeを使用する
- GitHub CopilotはエージェントモードでClaudeSonet4を使用する
- GitHub Copilotへのインプットとなる設計書、タスク(指示書)はマークダウンのテキスト形式でプロジェクト内に配置する
- タスクはフロントエンド、バックエンドで分割して作成し、それぞれの粒度は以下とする
	- フロントエンド = 1ユースケース
	- バックエンド = 1API
- プログラミング言語はTypeScriptを使用し、フロントエンドはVue.js、バックエンドはExpress、インフラはAWSを使用する
- フロントエンドの画面イメージはFigmaで作成済み
- アプリケーションを実行するのに必要なインフラはすでにAWS上に構築済み
- インプットとして使用する設計書
	- バックエンド
		- openapi定義
		- ER図
		- ドメインモデル図
		- ドメインルール
	- フロントエンド
		- Figmaで作成した画面イメージ(MCP経由で取得)
		- 画面設計書
- GitHub Copilotが出力するのはアプリケーションに関するソースコード、テストコード

# 課題
- GitHub Copilotへのインプットとなるタスクの最適な記載内容、構成、フォーマットの検討が必要
- GitHub Copilotへのインプットとなる設計書の最適な記述内容、構成、フォーマットの検討が必要
- 他に必要な設計情報は何かないか検討が必要
- フォルダ構成やインプットの読み込ませ方など、具体的な開発作業の進め方について検討が必要
- GitHub Copilotへのプロンプト内容について検討が必要(.github/copilot-instructions.md等)

# 指示

課題を解決して目標を達成する計画を具体的に作成してください
